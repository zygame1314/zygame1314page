Vercel 是一个流行的前端项目托管平台，本文将简单介绍如何配置 vercel.json 文件来实现项目的完美部署。

项目结构概览

首先来看看一个典型的，也是本网站的前端项目结构：


个人主页
├─ api/
├─ css/
├─ images/
├─ js/
└─ vercel.json

vercel.json 配置详解

下面我们将逐一解析配置文件中的每个部分：

1. 版本声明


{
    "version": 2
}

version: 2 表示使用 Vercel 平台的第二代部署系统，这目前推荐的版本。

2. builds 构建配置


{
    "builds": [
        {
            "src": "**/*.{html,css,js,jpg,png,svg,webp}",
            "use": "@vercel/static"
        },
        {
            "src": "api/**/*.js",
            "use": "@vercel/node"
        }
    ]
}

构建配置说明：

  • src: 使用 glob 模式匹配需要构建的文件
    • ** 表示匹配任意目录深度
    • *.{} 表示匹配指定的文件类型
  • use: 指定构建器
    • @vercel/static: 用于处理静态文件
    • @vercel/node: 用于处理 Node.js 服务端代码

3. routes 路由配置


{
    "routes": [
        {
            "src": "/api/(.*)",
            "dest": "/api/$1.js",
            "headers": {
                "Access-Control-Allow-Origin": "*",
                "Access-Control-Allow-Methods": "GET,POST,PUT,DELETE,OPTIONS",
                "Access-Control-Allow-Headers": "X-Requested-With,Content-Type"
            }
        },
        {
            "src": "/(.*)",
            "dest": "/$1",
            "continue": true
        }
    ]
}

路由配置说明：

  • src: 请求路径的匹配模式
    • (.*) 是正则表达式，用于捕获任意路径
    • 捕获范围会根据资源类型不同而不同：
      • 动态资源（如API）：捕获不含后缀名（因为在builds中已定义为.js）
      • 静态资源：捕获包含完整文件名和后缀
  • dest: 请求转发的目标路径
    • $1 表示引用第一个捕获组的内容
    • 后缀处理说明：
      • API路由：需要手动添加后缀 "/api/$1.js"
      • 静态资源：无需添加后缀 "/static/$1"
      • SPA重定向：使用 "/" 会自动解析到 index.html
  • headers: 设置响应头
    • 用于配置 CORS 跨域访问权限
    • 可以设置缓存策略等
  • continue: 是否继续匹配后续规则

进阶配置示例


{
    "version": 2,
    "builds": [
        {
            "src": "package.json",
            "use": "@vercel/static-build",
            "config": {
                "distDir": "build"
            }
        }
    ],
    "env": {
        "API_KEY": "@my-api-key"
    },
    "github": {
        "silent": true
    }
}

进阶特性说明：

  • env: 环境变量配置
  • github: GitHub 集成配置
  • distDir: 指定构建输出目录

配置说明

  1. builds 配置

    定义了如何构建不同类型的文件：

    • 静态文件使用 @vercel/static
    • API 接口使用 @vercel/node
  2. routes 配置

    定义了URL路由规则：

    • API 请求转发规则
    • 静态资源访问规则
    • CORS 跨域配置

完整示例


{
    "version": 2,
    "builds": [
        {
            "src": "**/*.{html,htm,css,js,jsx,ts,tsx……}", # 尽可能多地列出静态资源类型
            "use": "@vercel/static"
        },
        {
            "src": "api/**/*.js",
            "use": "@vercel/node"
        }
    ],
    "routes": [
        {
            "src": "/api/(.*)",
            "dest": "/api/$1.js",
            "headers": {
                "Access-Control-Allow-Origin": "*",
                "Access-Control-Allow-Methods": "GET, POST, OPTIONS",
                "Access-Control-Allow-Headers": "Content-Type"
            }
        },
        {
            "src": "/article/(.*)",
            "dest": "/"
        }
    ]
}

常见问题解决

注意事项

  • 尽可能确保所有静态资源类型都在 builds 配置中列出
  • API 路由配置要与实际目录结构匹配
  • 需要跨域访问时要配置正确的 headers

总结

合理配置 vercel.json 文件可以帮助我们更好地部署和管理项目。建议根据实际项目需求调整配置参数。
